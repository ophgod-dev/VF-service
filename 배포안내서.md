# 시야장애 자동분석 진단서 — 배포 안내서

Vercel을 처음 쓰셔도 따라만 하면 됩니다. 총 3단계, 약 20분.

전체 그림:
- **홈페이지(index.html)** — 병원에서 결과지를 올리는 화면
- **서버(api/analyze.js)** — GPT-5.5로 dB를 판독. **OpenAI 키는 여기에만** 숨겨집니다
- 계산(반경·시야점수·상실률)은 브라우저 안에서 검증된 코드로 수행됩니다

---

## 1단계 · OpenAI API 키 발급 (약 5분)

1. https://platform.openai.com 접속 → 로그인/가입
2. 우측 상단 **Settings → Billing** 에서 결제수단 등록 후 크레딧 충전 (예: $10)
   - 결과지 1장 판독은 몇십 원 수준입니다.
3. 좌측 **API keys → Create new secret key** → 키 복사 (`sk-...`로 시작, **한 번만 보이니 안전한 곳에 저장**)

> 이 키는 곧 Vercel에만 넣고, 홈페이지 코드에는 절대 넣지 않습니다.

---

## 2단계 · 코드를 GitHub에 올리기 (약 10분)

가장 쉬운 방법은 GitHub을 거치는 것입니다.

1. https://github.com 가입/로그인 → 우측 상단 **+ → New repository**
   - 이름: 예 `vf-service` → **Create repository**
2. 새 저장소 화면에서 **uploading an existing file** 링크 클릭
3. 이 폴더의 파일들을 **폴더 구조 그대로** 끌어다 놓습니다:
   ```
   index.html
   vercel.json
   package.json
   api/analyze.js      ← api 폴더 안에 있어야 함
   ```
   - `api` 폴더째로 드래그하면 구조가 유지됩니다. (GitHub 업로드 시 `api/analyze.js` 경로가 살아있는지 꼭 확인)
4. 아래 **Commit changes** 클릭

> GitHub이 어려우면 대안: 컴퓨터에 [Vercel CLI](https://vercel.com/docs/cli)를 설치해 폴더에서 `vercel` 명령으로 바로 올릴 수도 있습니다. 하지만 처음이면 GitHub 방식이 쉽습니다.

---

## 3단계 · Vercel에 배포하고 키 넣기 (약 5분)

1. https://vercel.com 접속 → **GitHub 계정으로 로그인(Continue with GitHub)**
2. **Add New… → Project** → 방금 만든 저장소(`vf-service`) 옆 **Import**
3. 배포 설정 화면에서 **Environment Variables**(환경 변수)를 펼쳐 아래를 추가:
   - Name: `OPENAI_API_KEY` / Value: 1단계에서 복사한 `sk-...` 키
   - (선택) Name: `OPENAI_MODEL` / Value: `gpt-5.5`  ← 넣지 않으면 자동으로 `gpt-5.5` 사용
4. **Deploy** 클릭 → 1~2분 기다리면 완료
5. 나오는 주소(예 `https://vf-service.vercel.app`)를 열면 홈페이지가 뜹니다

끝입니다. 이제 결과지를 올리고 **AI 자동 분석**을 누르면 dB가 판독되고 시야점수·상실률·진단서가 자동으로 나옵니다.

---

## 사용 방법

1. 우안(OD)·좌안(OS) 험프리 30-2 결과지 이미지를 각각 업로드
2. **AI 자동 분석** 클릭 → dB 격자가 채워짐
3. **격자에서 원본과 대조**하여 오독을 수정 (진단서 발급 전 필수)
4. 자각시력·인적사항·병명 입력 → 진단서 자동 완성
5. **인쇄/PDF** 또는 **Word(.docx) 저장**

## 자주 묻는 것

**Q. 키가 노출되지 않나요?**
네. 키는 Vercel 서버(환경변수)에만 있고, 홈페이지·브라우저·GitHub 어디에도 나타나지 않습니다. 브라우저는 이미지를 서버에 보내고, 서버가 대신 OpenAI를 호출합니다.

**Q. "OPENAI_API_KEY가 설정되지 않았습니다" 오류가 떠요.**
Vercel 프로젝트 → Settings → Environment Variables 에 `OPENAI_API_KEY`가 있는지 확인하고, 추가/수정했다면 **Deployments → 최신 배포 → Redeploy**로 재배포하세요.

**Q. 모델 이름을 바꾸려면?**
Vercel 환경변수 `OPENAI_MODEL` 값을 원하는 모델명으로 바꾸고 재배포하면 됩니다.

**Q. 판독이 가끔 틀려요.**
저해상도·기울어진 스캔에서 오독이 날 수 있습니다. 그래서 격자를 항상 **수정 가능**하게 두었습니다. 발급 전 원본과 대조가 원칙입니다. 결과지를 반듯하게, 수치 플롯이 선명하게 나오도록 캡처하면 정확도가 올라갑니다.

**Q. 나중에 결제(회당 1만원) 기능을 붙이려면?**
`api/analyze.js` 앞단에 결제 검증을 추가하고, 프론트에 결제 버튼을 넣으면 됩니다. 준비되면 그 부분을 만들어 드릴 수 있습니다.

## 개인정보 주의

결과지를 업로드하면 이미지가 OpenAI 서버로 전송됩니다. **환자 성명·등록번호가 보이지 않도록 잘라서** 수치 플롯만 업로드하세요. 서버 코드는 이미지를 저장하지 않습니다(판독 후 즉시 폐기).
